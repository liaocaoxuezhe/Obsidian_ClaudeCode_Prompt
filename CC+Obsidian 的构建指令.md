# 任务
从零开始构建我的 Obsidian 知识管理文件夹

# 需要创建以下内容

## 创建以下文件夹
- `/Clippings/`：保存的网页内容和文章
- `/Excalidraw/`：图表和视觉笔记
-  `/Zettelkasten/`：卡片和笔记法中的 zettelkasten，用来存放原子笔记
- `/template/`：笔记模板
- `/Diary/`：Obsidian 的每日文件
-  `/Create/`：维护创作的内容
- `/BASE/`：存放了一些现有的 BASE 数据库

## 为部分文件夹创建模板

模板要求如下

### Zettelkasten模板

  用途：用于创建原子化笔记，遵循 Zettelkasten 方法论

  Frontmatter 属性：
  - 创建时间: 使用 Templater 动态生成 ISO 格式时间
  - 随机标识符: 7位随机数字标识
  - 时间戳UID: 基于时间戳的唯一标识符
  - 页面ID: 组合格式ID（页码.随机数）

  内容结构：
  - 动态标题：<% tp.file.title %>
  - 空白内容区域，用于填写核心想法

### Create模板.md

  用途：用于创作内容模板，如文章、博客等

  Frontmatter 属性：
  - Status: 创作状态（默认"创作中"）
  - platform: 发布平台数组（默认为空）
  - tags: 标签数组（默认为空）
  - 创建时间/created: 双重时间标记
  - AREA: 领域分类
  - category: 分类
  - Date: 日期格式

  内容结构：
  - 动态标题：<% tp.file.title %>
  - 正文内容

### Diary 模板.md

  用途：用于每日复盘和笔记分析

 Frontmatter 属性：
  - Date: 日期格式

  结构化日记格式：
    a. 日期记录
    b. 新增笔记列表
    c. 当日笔记分析区域


## 创建 BASE 数据看板
1. Reference_base.base - 参考资料数据库

  主要逻辑：
  - 目标文件夹：专门筛选 Reference 文件夹中的文件
  - 视图类型：单一表格视图
  - 排序方式：按创建时间降序排列（最新的在前）
  - 显示字段：文件名、创建时间、修改时间、AI 摘要

  功能特点：
  - 用于管理参考资料文档
  - 按时间倒序展示，便于快速找到最新添加的参考资料
  - 包含 AI 摘要功能，方便快速了解文档内容

  2. Zettelkasten_base.base - 卡片盒数据库

  主要逻辑：
  - 目标文件夹：专门筛选 Zettelkasten 文件夹中的 Markdown 文件
  - 双视图设计：表格视图 + 卡片视图
  - 特殊功能：包含"随时间变化的随机数"公式

  核心功能：

  1. 表格视图：

    - 按创建时间降序，页面 ID 升序排列
    - 显示文件名、创建时间、标签、时间戳 UID
    - 适合结构化浏览和查找
  2. 卡片视图：

    - 使用随机数排序，实现随机展示效果
    - 卡片式布局，适合视觉化浏览
    - 随机排序算法基于时间戳 UID 和当前时间分钟数

  随机数公式解析：
  随时间变化的随机数: (( (number(this.时间戳 UID) * 100011979 + 500067713) %
  900066731 ) * ( (now().minute * 800067089 + 800068411) % 800053967 ) + 900067309) %
   900066571

  这个公式的作用是：
  - 结合笔记的时间戳 UID 和当前时间的分钟数
  - 通过复杂的数学运算生成一个随时间变化的随机数
  - 实现卡片视图的随机排序，便于发现意外的知识连接

！！！ 创建 base 时一定要仔细阅读 BASE 的相关说明，按照争取地格式创建 .base 文件

  设计理念：
  Zettelkasten 数据库的核心是促进知识的随机碰撞和意外发现，通过随机排序帮助用户打破思
  维定式，发现笔记之间意想不到的联系。
